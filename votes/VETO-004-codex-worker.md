## Teammate: codex-worker
## 안건: VETO-004
## 근거
`GEMINI.md`와 `AGENTS.md`의 본문은 실제로 agent name 한 줄만 다르며 중복 자체는 맞다. 다만 현재 제안은 "단일 템플릿 파일로 통합"이 아니라 "런타임 동적 주입 체계 추가"가 핵심 변화다. 이 부분은 비용 대비 이득이 작다.

첫째, 두 파일은 각 CLI가 정적으로 인식하는 진입 파일 성격이 강하다. 현재는 저장소에 파일 두 개가 존재하면 끝이지만, 제안안은 셸 스크립트가 실행 시점마다 템플릿 해석과 agent name 주입을 정확히 수행해야 한다. 즉 문서 중복 1줄을 줄이는 대신 런타임 의존성과 실패 지점을 추가한다.

둘째, agent identity는 프로토콜의 일부다. `clmux.zsh`와 `bridge-mcp-server.js` 전반에서 `gemini-worker`, `codex-worker`가 기본값과 라우팅 키로 쓰이고 있다. 이 상태에서 지시 파일까지 동적으로 만들면, 실제 실행 agent 이름과 지시 파일 내 이름이 어긋나는 클래스의 버그가 새로 생긴다.

셋째, 중복 제거 효과가 매우 제한적이다. 현재 중복은 20줄 미만 정적 문서 2개이며, 변경 빈도도 낮다. 반면 쉘 주입 방식은 각 CLI별 전달 메커니즘 차이까지 고려해야 하므로 유지보수성이 자동으로 좋아지지 않는다.
## 대안 (VETO 시)
런타임 주입 대신 정적 파일 유지가 맞다. 중복이 거슬리면 다음 두 가지 중 하나로 해결하는 편이 낫다.

1. `teammate-protocol.md`를 원문 source로 두고, 릴리스/검증 스크립트가 `GEMINI.md`와 `AGENTS.md`를 생성하게 한다. 사용자와 CLI는 계속 정적 파일만 본다.
2. 더 단순하게는 두 파일을 그대로 유지하되, CI나 로컬 체크 스크립트로 agent name 한 줄을 제외한 내용 동일성을 검증한다.
